## 7) What to avoid

- Do not fill the README with too many badges and random icons
- Do not add fake statistics or misleading “years of experience” claims
- Do not pin tutorial repos or unfinished experiments
- Do not put every technology you ever touched
- Do not make GitHub read like LinkedIn copy-paste only; show projects

## 9) Strong recommendation

If you want GitHub to function like a CV, the winning formula is:

- **Professional profile settings**
- **A concise profile README**
- **High-quality pinned repositories**
- **At least 2 to 3 public repos that look production-ready**

Without public repos, GitHub will not help you much in hiring.
